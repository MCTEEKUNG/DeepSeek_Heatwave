คำถามใหญ่: "สกิล +0.168 จริง หรือฟลุ๊ก?"

รอบที่แล้วเราได้ผลว่าโมเดล logistic ทำคะแนน BSS = +0.168 ที่ lead 2 สัปดาห์ — ดูเหมือนชนะ baseline แต่ "ดูเหมือน" ยังไม่พอ งานรอบนี้คือพิสูจน์ว่ามันแกร่งและเชื่อถือได้จริง

BSS (Brier Skill Score) คืออะไร?

คะแนนวัดว่าโมเดลทำนาย "ความน่าจะเป็น" ได้ดีกว่า baseline (การเดาแบบง่าย ๆ ตามค่าเฉลี่ยฤดูกาล) แค่ไหน:
BSS > 0 = ดีกว่า baseline = 0 = เท่า baseline < 0 = แย่กว่า baseline

สูตร: BSS = 1 − (ความคลาดเคลื่อนของโมเดล ÷ ความคลาดเคลื่อนของ baseline)

เฟส 1 · ความไม่แน่นอน

วัด "ความมั่นใจ" ด้วย Block Bootstrap

ตัวเลข +0.168 มาจากข้อมูลชุดเดียว ถ้าเราโชคดี/โชคร้ายล่ะ? เราจึง "สุ่มซ้ำ" ข้อมูลหลายพันครั้ง เพื่อดูว่าคะแนนแกว่งแค่ไหน → ได้ ช่วงความเชื่อมั่น 95% (CI)

ทำมายังไง?

ข้อมูลพยากรณ์เป็น "อนุกรมเวลา" วันที่ติดกันสัมพันธ์กันสูง → สุ่มทีละวันไม่ได้ (จะทำลายความสัมพันธ์) เราจึงสุ่มเป็น "ก้อนวันติดกัน" ยาว 28 วัน (moving-block bootstrap) ทำซ้ำ 2,000 รอบ

▲ จุดส้ม = ก้อน 28 วันที่ถูกสุ่มในแต่ละรอบ

แปลว่าอะไร?

ถ้าช่วง CI ไม่คร่อมเลข 0 = มั่นใจว่าชนะ baseline จริง ไม่ใช่ฟลุ๊ก

ถ้า CI คร่อม 0 = ยังสรุปไม่ได้ อาจเป็นแค่ความบังเอิญ

เฟส 2 · แยกตามสภาวะอากาศ

ENSO: ตัวขับเคลื่อนที่พลิกความเข้าใจเดิม

เราแบ่ง "วันออกพยากรณ์" ตามสภาวะมหาสมุทรแปซิฟิก (เอลนีโญ / ปกติ / ลานีญา) จากค่า Niño3.4 ที่โมเดลเห็นจริง แล้วดูว่าโมเดลทำงานดีในสภาวะไหน

ทำมายังไง?

🔑 แก้ความเข้าใจผิดเดิม: "ปีเอลนีโญอ่อน"

ความจริงตรงข้าม — ช่วงเอลนีโญโมเดลแกร่งสุด เพราะ climatology (เฉลี่ยทุกปี) ทายต่ำไปในปีเอลนีโญ โมเดลที่รู้จัก ENSO เลยเอาชนะได้เยอะ. ส่วน "fold ที่ดูอ่อน" จริง ๆ คือช่วง 2016–18 ที่ ไม่มีคลื่นความร้อนเลย (base rate = 0.000) ทำให้คะแนน BSS เพี้ยน (−0.733) — เป็น artifact ของช่วงไร้เหตุการณ์ ไม่ใช่ความอ่อนของโมเดล

อ่านยังไง?

ฤดูกาลคือตัวทำนายหลัก (คลื่นความร้อนมาตามฤดู) ตามด้วย ความชื้นในดิน (soil moisture) — ยืนยันสมมติฐาน land–atmosphere coupling ของวิทยานิพนธ์

บทสรุป

5 สิ่งที่ได้เรียนรู้

1. สกิลมีจริงและมีนัยสำคัญที่ lead 2–4 สัปดาห์ (ช่วงสั้น) แล้วจางเมื่อ lead ยาว
2. โมเดลที่ควรใช้จริงคือ logistic_balanced_cal (ชนะแม้ persistence) ไม่ใช่ logistic ดิบ
3. ENSO เป็นตัวขับเคลื่อนหลัก — ปีเอลนีโญคลื่นความร้อนถี่ขึ้น โมเดลทำงานดีสุด (ไม่ใช่อ่อน)
4. LightGBM ต้อง recalibrate เสมอ (ปัญหาอยู่ที่ calibration ไม่ใช่การแยกแยะ)
5. ความชื้นในดินยืนยัน land–atmosphere coupling · linear model อาจดีขึ้นถ้าลดฟีเจอร์ซ้ำซ้อน

ทำไมเราเชื่อผลนี้ได้?

ทุกตัวเลขมาจากวิธีที่ กันการโกง: แบ่งข้อมูลตามเวลา (ไม่ให้เห็นอนาคต), block bootstrap คุมความไม่แน่นอน, คุม false discovery (BH-FDR), และตรวจ "full reproduce ตัวเลขเดิมเป๊ะ" ทุกการทดลอง
