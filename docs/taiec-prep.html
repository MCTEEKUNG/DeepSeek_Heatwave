01

คณิตศาสตร์สำหรับ AI

ROUND 1

AI ทั้งหมดสร้างจากคณิต 4 ก้อน: ฟังก์ชัน · เมทริกซ์ · ความน่าจะเป็น · ความชัน ถ้าเข้าใจว่าแต่ละก้อน "ไปโผล่ตรงไหนใน AI" จะจำได้โดยไม่ต้องท่อง

สมการเชิงเส้น & ฟังก์ชัน

ฟังก์ชันเชิงเส้น y = wx + b คือโมเดล AI ที่เรียบง่ายที่สุด — w (weight) คือความชัน บอกว่า x มีผลต่อ y แค่ไหน, b (bias) คือจุดตัดแกน

ŷ = w₁x₁ + w₂x₂ + ... + wₙxₙ + bLinear Regression = สมการเส้นตรงหลายตัวแปร — "การเรียนรู้" คือการหา w, b ที่ทำให้ทายแม่นสุด
  • ต้องคล่อง: แก้ระบบสมการ 2–3 ตัวแปร, หาความชันจาก 2 จุด
  • ฟังก์ชันไม่เชิงเส้นที่ AI ใช้: sigmoid บีบค่าให้อยู่ 0–1 (กลายเป็นความน่าจะเป็น), ReLU = max(0, x)
  • composite function f(g(x)) = หัวใจของ neural network หลายชั้น
ไปโผล่ที่: ทุกโมเดล — neural network 1 neuron ก็คือ wx + b แล้วครอบด้วยฟังก์ชันไม่เชิงเส้น

เมทริกซ์ & เวกเตอร์

เวกเตอร์ = ลิสต์ตัวเลข 1 แถว (ข้อมูล 1 ตัวอย่าง) · เมทริกซ์ = ตาราง (ข้อมูลทั้งชุด หรือ weight ทั้งชั้นของ neural net)

(A·B)ᵢⱼ = Σₖ Aᵢₖ Bₖⱼคูณเมทริกซ์: แถวของ A จับคู่คอลัมน์ของ B — คูณทีละคู่แล้วบวก (dot product)
  • เงื่อนไขคูณได้: (m×n)·(n×p) → ได้ (m×p) — ตัวในต้องเท่ากัน
  • สำคัญ: A·B ≠ B·A (ไม่สลับที่!) — ออกสอบบ่อย
  • Identity matrix I: คูณอะไรได้ตัวเดิม (เหมือนเลข 1)
  • Transpose Aᵀ: พลิกแถวเป็นคอลัมน์
  • Dot product ของ 2 เวกเตอร์ = ตัววัด "ความคล้าย" (similarity)
ไปโผล่ที่: 1 ชั้นของ neural network = การคูณเมทริกซ์ 1 ครั้ง (output = W·x + b) · ระบบแนะนำ/ search ใช้ dot product วัดความคล้าย

ความน่าจะเป็น & สถิติ

AI สมัยใหม่ตอบเป็น "ความน่าจะเป็น" ไม่ใช่ "ใช่/ไม่ใช่" — เพราะโลกจริงไม่แน่นอน

P(A|B) = P(B|A)·P(A) / P(B)Bayes' theorem — กลับทิศเงื่อนไข: รู้ P(อาการ|โรค) → หา P(โรค|อาการ)
  • P(A และ B) = P(A)·P(B) เมื่ออิสระต่อกันเท่านั้น
  • P(A หรือ B) = P(A) + P(B) − P(A และ B)
  • Conditional probability P(A|B) = P(A∩B)/P(B)
  • ค่าคาดหวัง E[X] = Σ xᵢ·P(xᵢ) — "ค่าเฉลี่ยถ่วงด้วยโอกาส"
  • mean / median / SD — SD บอกการกระจาย, median ทนค่า outlier
ไปโผล่ที่: spam filter (Naive Bayes), LLM ทายคำถัดไปด้วยการแจกแจงความน่าจะเป็น, โมเดลพยากรณ์ heatwave ก็ตอบเป็น P(เกิด heatwave)

ความชัน → Gradient Descent

อนุพันธ์ (derivative) = ความชันของฟังก์ชัน ณ จุดหนึ่ง — บอกว่า "ถ้าขยับ x นิดเดียว y เปลี่ยนเท่าไหร่และไปทางไหน"

w ← w − α · ∂L/∂wกฎอัปเดตของ gradient descent: เดินสวนทางความชันของ Loss ทีละก้าวขนาด α (learning rate)
  • Loss function L = ตัววัด "ทายพลาดแค่ไหน" เช่น MSE = ค่าเฉลี่ยของ (จริง−ทาย)²
  • ความชันเป็นบวก → ลด w · ความชันเป็นลบ → เพิ่ม w → เดินลงเขาเสมอ
  • α ใหญ่ไป = กระโดดข้ามก้นเหว · α เล็กไป = เดินช้ามาก (ลองเล่นใน demo โมดูล 03)
  • จุดต่ำสุด = ความชัน 0 → โมเดล "เรียนรู้เสร็จ"
ไปโผล่ที่: การ train ทุกโมเดลตั้งแต่ linear regression ยัน GPT คือ gradient descent บน loss ทั้งนั้น
QUIZ // ลองของ — โมดูล 01
Q1 — เมทริกซ์ A ขนาด 2×3 คูณกับ B ขนาด 3×4 ผลลัพธ์มีขนาดเท่าไร?
เฉลย: 2×4 — (m×n)·(n×p) = (m×p) ตัวใน (3=3) ต้องเท่ากันถึงคูณได้ แล้วเอาตัวนอกมาเป็นขนาดผลลัพธ์
Q2 — โยนลูกเต๋า 2 ลูก ความน่าจะเป็นที่ผลรวม = 7 คือ?
เฉลย: 1/6 — ผลลัพธ์ทั้งหมด 36 แบบ, รวมเป็น 7 ได้ 6 แบบ (1+6, 2+5, 3+4, 4+3, 5+2, 6+1) → 6/36 = 1/6
Q3 — ถ้าความชันของ Loss เทียบกับ w เป็นค่า "ลบ" gradient descent จะทำอะไร?
เฉลย: เพิ่ม w — กฎคือ w ← w − α·(ความชัน) ความชันลบ → ลบด้วยค่าลบ = บวก → w เพิ่มขึ้น เดินลงเขาเข้าหาจุดต่ำสุด
02

Programming & Problem-Solving

ROUND 1 + 2

ภาษาหลักของสนามนี้คือ Python — รอบ 1 วัดความเข้าใจโค้ดและอัลกอริทึม รอบ 2 วัดว่าเขียนจริงได้เร็วแค่ไหน

Python ที่ต้องคล่องเหมือนหายใจ

  • โครงสร้างข้อมูล 4 ตัว: list (ลำดับ, แก้ได้) · tuple (แก้ไม่ได้) · dict (key→value, ค้นเร็ว O(1)) · set (ไม่ซ้ำ, เช็คสมาชิกเร็ว)
  • List comprehension: [x**2 for x in range(10) if x % 2 == 0]
  • Slicing: a[1:4], a[::-1] (กลับลำดับ)
  • ฟังก์ชัน: default args, *args, return หลายค่า
  • อ่านโค้ดแล้ว "ไล่ค่า" ในหัวได้ — ข้อสอบชอบถาม output
# ข้อสอบคลาสสิก: ไล่ค่าให้ทัน
def f(n):
    return n * f(n-1) if n > 1 else 1
print(f(5))  # 120 — recursion หา factorial

OOP — Object-Oriented Programming

แนวคิด 4 เสา ที่ออกสอบแน่:

  • Class/Object: class = พิมพ์เขียว, object = ของจริงที่สร้างจากพิมพ์เขียว
  • Encapsulation: ซ่อนข้อมูลภายใน เข้าถึงผ่าน method
  • Inheritance: class ลูกได้ความสามารถ class แม่ (class Dog(Animal))
  • Polymorphism: method ชื่อเดียวกัน ทำงานต่างกันตาม class (override)
class Model:
    def predict(self, x): return "base"

class HeatwaveModel(Model):   # inheritance
    def predict(self, x):       # polymorphism
        return "P(heatwave) = 0.73"
เชื่อมโยง: library ML จริง (scikit-learn) ใช้ OOP เป๊ะ ๆ — ทุกโมเดลมี .fit() / .predict() เหมือนกัน (polymorphism ของจริง)

อัลกอริทึม & Big-O

Big-O = "เวลาทำงานโตเร็วแค่ไหนเมื่อข้อมูลใหญ่ขึ้น" — เรียงจากเร็วไปช้า:

Big-Oตัวอย่างn = 1 ล้าน
O(1)ดึงจาก dict / array indexทันที
O(log n)binary search (ข้อมูลเรียงแล้ว)~20 ครั้ง
O(n)วนลูปครั้งเดียว / linear search1 ล้านครั้ง
O(n log n)sort ที่ดี (merge sort, Timsort)~20 ล้าน
O(n²)ลูปซ้อนลูป / bubble sort1 ล้านล้าน 💀
กับดักข้อสอบ: ลูปซ้อน 2 ชั้นบน n เดียวกัน = O(n²) · binary search ใช้ได้เฉพาะข้อมูลที่ "เรียงแล้ว" เท่านั้น

สูตรแก้โจทย์ Problem-Solving

เจอโจทย์ที่ไม่เคยเห็น ใช้ลำดับนี้เสมอ:

  • 1. แปลงโจทย์ → input คืออะไร output คืออะไร เขียนตัวอย่างเอง 1 เคส
  • 2. เคสมุม (edge cases) → ลิสต์ว่าง? ค่าติดลบ? ข้อมูลซ้ำ? n=0?
  • 3. brute force ก่อน → วิธีโง่ที่สุดที่ถูกต้อง แล้วค่อยถามว่าเร็วขึ้นได้ไหม
  • 4. รู้จัก pattern: two pointers, hash map (dict) นับความถี่, sliding window, recursion
  • 5. ไล่โค้ดด้วยมือ กับตัวอย่างก่อนส่ง

QUIZ // ลองของ — โมดูล 02
Q1 — a = [1,2,3,4,5]; print(a[1:4:2]) ได้อะไร?
เฉลย: [2, 4] — slice [start:stop:step] = เริ่ม index 1, หยุดก่อน index 4, กระโดดทีละ 2 → index 1 และ 3 → ค่า 2 และ 4
Q2 — ต้องเช็คว่า "คำนี้เคยเจอหรือยัง" ในข้อมูลล้านคำ ควรใช้โครงสร้างไหนให้เร็วสุด?
เฉลย: set — เช็คสมาชิกใน set เฉลี่ย O(1) ส่วน list ต้องไล่ O(n), binary search ก็ O(log n) แถมต้องเรียงก่อน
Q3 — ลูปซ้อน 2 ชั้น ชั้นนอกวน n รอบ ชั้นในวน n รอบ ความซับซ้อนคือ?
เฉลย: O(n²) — ทุกรอบของชั้นนอก ชั้นในวนเต็ม n รอบ → n×n ครั้ง (O(2n) ไม่มีอยู่จริง — ค่าคงที่ตัดทิ้ง เหลือ O(n))
03

AI Fundamentals

ROUND 1 หัวใจหลัก

โครงใหญ่: AI ⊃ Machine Learning ⊃ Deep Learning — AI คือเป้าหมาย (เครื่องฉลาด), ML คือวิธี (เรียนจากข้อมูลแทนการเขียนกฎ), DL คือ ML ที่ใช้ neural network หลายชั้น

3 โหมดการเรียนรู้

โหมดข้อมูลตัวอย่าง
Supervisedมีเฉลย (label)ทายราคาบ้าน, จำแนกสแปม
Unsupervisedไม่มีเฉลยจัดกลุ่มลูกค้า (clustering)
Reinforcementเรียนจาก rewardAI เล่นเกม, หุ่นยนต์

Supervised แบ่งย่อยอีก 2: Regression (ทายตัวเลข เช่น อุณหภูมิ) vs Classification (ทายหมวด เช่น heatwave / ไม่ heatwave)

กับดักข้อสอบ: "ทายความน่าจะเป็นของเหตุการณ์" = classification (logistic regression) ไม่ใช่ regression แม้ชื่อจะมีคำว่า regression!

โมเดลพื้นฐานที่ต้องรู้จัก

  • Linear Regression — ลากเส้นตรงให้ fit จุด, ทายตัวเลข
  • Logistic Regression — เส้นตรงครอบ sigmoid → ได้ความน่าจะเป็น 0–1
  • Decision Tree — ต้นไม้คำถาม if/else, อ่านง่าย แต่ overfit ง่าย
  • Random Forest / Gradient Boosting — รวมต้นไม้หลายต้นโหวตกัน แม่นขึ้นมาก
  • k-NN — ดูเพื่อนบ้าน k ตัวที่ใกล้สุดแล้วโหวต
  • k-Means — จัดกลุ่ม (unsupervised) จับจุดเข้า k กลุ่มตามระยะ
  • Neural Network — ชั้นของ (เมทริกซ์คูณ + ฟังก์ชันไม่เชิงเส้น) ซ้อนกัน

Overfitting — ศัตรูหมายเลขหนึ่ง

Overfit = ท่องข้อสอบเก่าจนจำเฉลยได้ แต่ทำข้อสอบใหม่ไม่ได้ (คะแนน train สูง, test ต่ำ) · Underfit = โมเดลง่ายไป ทำได้แย่ทั้งคู่

  • ทำไมต้อง split: train (เรียน) / validation (จูน) / test (วัดจริง ห้ามแตะจนจบ)
  • Cross-validation: แบ่ง k ส่วน ผลัดกันเป็น test → ค่าเฉลี่ยน่าเชื่อกว่า
  • แก้ overfit: เพิ่มข้อมูล, ลดความซับซ้อนโมเดล, regularization (ลงโทษ w ที่ใหญ่), early stopping
  • ข้อมูล time series ห้าม split สุ่ม! ต้องตัดตามเวลา (train อดีต → test อนาคต) ไม่งั้นโมเดล "แอบเห็นอนาคต" = data leakage
ออกสอบชัวร์: ให้กราฟ train loss ลด แต่ validation loss เริ่มเด้งขึ้น → จุดนั้นคือเริ่ม overfit ควรหยุด (early stopping)

วัดผลโมเดล — Metrics

Confusion matrix ของ classification (ทาย heatwave):

ทาย: เกิดทาย: ไม่เกิด
จริง: เกิดTP ✓FN ✗ (พลาดเตือน!)
จริง: ไม่เกิดFP ✗ (เตือนหลอก)TN ✓
  • Accuracy = (TP+TN)/ทั้งหมด — หลอกได้ถ้าข้อมูล imbalanced: heatwave เกิดแค่ 5% ของวัน โมเดลทาย "ไม่เกิด" ตลอดก็ accuracy 95% แต่ไร้ค่า
  • Precision = TP/(TP+FP) — ที่เตือนไป จริงกี่ส่วน
  • Recall = TP/(TP+FN) — เหตุการณ์จริง จับได้กี่ส่วน
  • F1 = ค่าเฉลี่ยฮาร์มอนิกของ precision กับ recall
  • ROC-AUC = วัดการแยกชั้นทุก threshold · Brier score = วัดความแม่นของ "ความน่าจะเป็น" ที่ทาย

Neural Network ใน 60 วินาที

  • 1 neuron: รับ input หลายตัว → ถ่วงน้ำหนัก w → บวก b → ผ่าน activation (ReLU/sigmoid)
  • หลาย neuron เรียงเป็น layer, หลาย layer ซ้อน = deep learning
  • Forward pass: ข้อมูลไหลไปข้างหน้า → ได้คำทาย
  • Backpropagation: คำนวณว่าแต่ละ w มีส่วนทำให้พลาดเท่าไร (chain rule) → อัปเดตด้วย gradient descent
  • ชนิดสำคัญ: CNN (ภาพ — กรองหา pattern เฉพาะที่), RNN (ลำดับ), Transformer (ภาษา — กลไก attention)

Generative AI — ของใหม่ที่สอบแน่

  • LLM (เช่น GPT, Claude): โมเดล transformer ที่เรียนจากข้อความมหาศาล หลักการคือ "ทายโทเค็นถัดไป" ซ้ำ ๆ
  • Token = หน่วยย่อยของข้อความ (คำ/ส่วนของคำ) · Embedding = แปลงคำเป็นเวกเตอร์ตัวเลขที่คำความหมายใกล้กันอยู่ใกล้กัน
  • Attention = กลไกให้โมเดล "เพ่ง" คำที่เกี่ยวข้องกันแม้อยู่ไกลกันในประโยค
  • Hallucination = โมเดลตอบมั่นใจแต่ผิด — เพราะมันทายคำที่ "น่าจะมา" ไม่ใช่ค้นความจริง
  • Fine-tuning = เอาโมเดลที่ train แล้วมาเรียนต่อกับข้อมูลเฉพาะทาง · RAG = ให้โมเดลค้นเอกสารก่อนตอบ ลด hallucination
  • Diffusion model (สร้างภาพ): เรียนวิธี "ลบ noise" ทีละขั้นจนได้ภาพ

QUIZ // ลองของ — โมดูล 03
Q1 — ข้อมูล heatwave เกิดแค่ 5% ของวันทั้งหมด ทำไม accuracy ถึงเป็น metric ที่แย่?
เฉลย: นี่คือปัญหา class imbalance — ต้องดู precision/recall/F1 หรือ Brier/AUC แทน (recall สำคัญมากกับงานเตือนภัย เพราะ FN = พลาดเตือนเหตุการณ์จริง)
Q2 — train loss ลดลงเรื่อย ๆ แต่ validation loss เริ่มเพิ่มขึ้น แปลว่าอะไร?
เฉลย: overfitting — โมเดลเก่งขึ้นเฉพาะกับข้อมูลที่เคยเห็น แต่แย่ลงกับข้อมูลใหม่ ทางแก้: early stopping, regularization, เพิ่มข้อมูล
Q3 — ทำไมข้อมูลอนุกรมเวลา (time series) ห้ามแบ่ง train/test แบบสุ่ม?
เฉลย: data leakage — งานพยากรณ์จริงโมเดลรู้แค่อดีต ดังนั้นต้อง split ตามเวลา (เช่น train 1994–2017, test 2018–2023) หรือใช้ rolling-origin CV
Q4 — LLM อย่าง ChatGPT ทำงานด้วยหลักการหลักคืออะไร?
เฉลย: next-token prediction — และนี่คือเหตุผลที่เกิด hallucination ได้: โมเดลเลือกคำที่ "น่าจะมา" ตามสถิติ ไม่ได้ตรวจสอบความจริง
04

AI System Design

ROUND 2 อาวุธลับ

รอบชิงไม่ได้วัดว่า "รู้สูตรไหม" แต่วัดว่า ออกแบบระบบ AI ที่ใช้งานได้จริงเป็นไหม — นี่คือจุดที่คนทำโปรเจกต์จริงมาแล้วได้เปรียบที่สุด

ML Pipeline — 6 ขั้นที่ต้องท่องเป็นลำดับ

  • 1. Frame the problem — ทายอะไร? input/output คืออะไร? วัดความสำเร็จด้วยอะไร?
  • 2. Data — หาจากไหน, สะอาดไหม, มี bias ไหม, ต้อง validate (หน่วย/ช่วงค่า/missing)
  • 3. Features — แปลงข้อมูลดิบเป็นตัวแปรที่โมเดลกินได้ (normalize, encode, lag)
  • 4. Model — เริ่มจากง่ายสุดเสมอ (logistic/tree) ก่อนไปของซับซ้อน
  • 5. Evaluate — เทียบกับ baseline บน test set ที่ไม่เคยแตะ
  • 6. Deploy/Monitor — ใช้จริง + เฝ้าดูว่าข้อมูลโลกจริงเปลี่ยน (drift) ไหม

Baseline — กฎเหล็กที่กรรมการชอบ

โมเดล AI "ดี" ไม่ใช่เพราะคะแนนสูง แต่เพราะชนะตัวเปรียบเทียบที่ยุติธรรม:

  • Majority baseline: ทายคลาสที่เจอบ่อยสุดตลอด — โมเดลต้องชนะอันนี้ก่อน
  • Persistence: "พรุ่งนี้เหมือนวันนี้" — สำหรับ time series มันแข็งกว่าที่คิด
  • Climatology: ค่าเฉลี่ยตามฤดูกาล/ประวัติศาสตร์
  • ถ้าโมเดลซับซ้อนชนะ baseline แค่นิดเดียว → เลือก baseline (ง่ายกว่า ถูกกว่า อธิบายได้)
ประโยคทองตอน present: "โมเดลของผมชนะ baseline X เท่าไร วัดด้วย metric อะไร และทำไม metric นั้นเหมาะกับโจทย์นี้"

กับดักที่ทำให้ระบบ AI พังจริง

  • Data leakage: ข้อมูลที่ "ไม่ควรรู้ตอนทาย" หลุดเข้า training — เช่น normalize ด้วยสถิติของทั้งชุด (รวม test) หรือ feature ที่เกิดหลังเหตุการณ์
  • Train/serve skew: ตอน train ใช้ข้อมูลสะอาด ตอนใช้จริงข้อมูลสกปรก
  • Imbalanced data: ใช้ accuracy แล้วหลงดีใจ (โมดูล 03)
  • หน่วยผิด: Kelvin ปน Celsius, เมตรปนฟุต — ระบบจริงต้องมี "ด่านตรวจ" ที่ fail ดัง ๆ ทันที
  • ไม่มี error handling: ดาวน์โหลดล่มกลางทางแล้วเริ่มใหม่หมด — ระบบดีต้อง resume ได้

ตอบคำถาม System Design ให้เป็นระบบ

โจทย์แนว "จงออกแบบระบบ AI สำหรับ ____" ตอบตามโครงนี้:

  • ผู้ใช้ & ปัญหา → ใครใช้ ตัดสินใจอะไรจากผลลัพธ์
  • นิยาม input/output ชัด ๆ → รวมถึง "ความถี่" (เรียลไทม์? รายวัน?)
  • ข้อมูล → แหล่ง, ปริมาณ, ปัญหาที่คาด (missing, bias, privacy)
  • โมเดล → เริ่มง่าย + เหตุผลที่เลือก
  • การวัดผล → metric + baseline + วิธี split ที่ไม่ leak
  • ความเสี่ยง → ถ้าโมเดลทายผิดใครเดือดร้อน มี fallback ไหม
เคล็ดลับ: พูดถึง trade-off เสมอ ("แม่นขึ้นแต่ช้าลง", "recall สูงขึ้นแลกกับเตือนหลอกมากขึ้น") — นี่คือสิ่งที่แยกวิศวกรออกจากคนท่องจำ
CASE STUDY // โปรเจกต์ของคุณเอง = ตัวอย่างคำตอบชั้นดี

🔥 Sub-seasonal Heatwave Prediction (Thailand)

โปรเจกต์ heatwave ที่คุณทำอยู่ ตอบครบทุกข้อของ system design — ใช้เล่าในสนามและพอร์ตได้เลย:

  1. Frame: ทาย P(เกิด heatwave) ล่วงหน้า 2–6 สัปดาห์ — output เป็นความน่าจะเป็น ไม่ใช่ ใช่/ไม่ใช่ เพราะผู้ใช้ (สาธารณสุข/เกษตร) ต้องชั่งความเสี่ยงเอง
  2. นิยาม target ตรวจสอบได้: Tmax > เปอร์เซ็นไทล์ 90 ของภูมิอากาศฐาน ≥ 3 วันติด — ไม่ใช่นิยามลอย ๆ
  3. Data engineering จริง: ดึง ERA5 แบบ resume ได้ + ด่านตรวจหน่วย Kelvin→°C ที่ fail ดัง ๆ
  4. กัน leakage: ใช้ blocked time-series CV — train อดีต ทดสอบอนาคตเสมอ
  5. Baseline ยุติธรรม: ต้องชนะ climatology + persistence วัดด้วย Brier Skill Score, ROC-AUC, reliability diagram
  6. เริ่มจากโมเดลง่าย: logistic regression ก่อน แล้วค่อย gradient boosting — เทียบกันแบบ ablation
QUIZ // ลองของ — โมดูล 04
Q1 — คุณ normalize ข้อมูลด้วย mean/SD ที่คำนวณจาก "ข้อมูลทั้งหมด" ก่อนค่อยแบ่ง train/test ปัญหาคือ?
เฉลย: leakage — ที่ถูกคือ fit ตัว normalizer จาก train เท่านั้น แล้วเอาไป transform ทั้ง train และ test (หลักเดียวกับ scikit-learn: fit_transform(train) แล้ว transform(test))
Q2 — โมเดล deep learning ของคุณชนะ logistic regression อยู่ 0.2% แต่ช้ากว่า 50 เท่า ควรเลือกอะไรไป production?
เฉลย: วิศวกรรมคือ trade-off — ความง่าย ความเร็ว การอธิบายได้ มีมูลค่าจริง คำตอบแบบนี้แหละที่ทำให้กรรมการรู้ว่าคุณ "คิดแบบ engineer"

  • ข้อคำนวณ: เขียนทดเสมอ อย่าคิดในหัวล้วน — ความเครียดทำให้เลขง่ายพลาด
  • ข้อโค้ด: ไล่ค่าทีละบรรทัดบนกระดาษทด (ตาราง: ตัวแปร | ค่า)

รอบ 2 — Mini Project สด

  • กฎ 80/20: ใช้ 20% แรกของเวลาวางแผน (โจทย์ถามอะไรกันแน่ → pipeline 6 ขั้น) ก่อนแตะคีย์บอร์ด
